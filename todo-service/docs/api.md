# API Reference

This document provides a detailed description of the REST API exposed by the Spring Boot Todo service.

## Base URL
```
http://localhost:8081
```

## Data Model
| Field     | Type    | Description
|-----------|---------|-----------
| id        | Long    | Auto‑generated primary key.
| title     | String  | Short title of the Todo.
| details   | String? | Optional longer description.
| completed | Boolean | Flag indicating completion.

## Endpoints
| Method | Path                | Description                                           | Request Body (JSON) | Response Body (JSON) |
|--------|---------------------|-------------------------------------------------------|---------------------|----------------------|
| GET    | `/todos`           | Retrieve all todos.                                   | – | Array of Todo objects.
| GET    | `/todos/{id}`     | Retrieve a single todo by ID.                          | – | Todo object.
| POST   | `/todos`           | Create a new todo.                                     | `{"title":"...","details":"...","completed":false}` | Created Todo object.
| PATCH  | `/todos/{id}`     | Update an existing todo.                               | Partial Todo fields (`title?`, `details?`, `completed?`) | Updated Todo object.
| DELETE | `/todos/{id}`     | Delete a todo by ID.                                   | – | Empty body – `204 No Content`.

## Error Handling
All error responses follow the standard Spring `ResponseStatusException` mechanism.

Examples:
- **404 Not Found** – No todo with the requested ID.
- **400 Bad Request** – Malformed request body.

## CORS
The backend is preconfigured to accept requests from `http://localhost:4200`.
